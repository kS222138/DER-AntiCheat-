## 📝 Reporting Issues / 报告问题

**English** 🇬🇧
- 🔍 Please search existing issues first to avoid duplicates
- 📋 Describe clearly: what happened, what you expected, how to reproduce
- 💻 Include Godot version, OS, and plugin version

**中文** 🇨🇳
- 🔍 请先搜索已有的 Issues，避免重复
- 📋 描述清楚：发生了什么、预期是什么、如何复现
- 💻 附上 Godot 版本、操作系统、插件版本

## 💻 Submitting Code / 提交代码

**English** 🇬🇧
1. 🍴 Fork this repository
2. 🌿 Create your feature branch (`git checkout -b feature/amazing-feature`)
3. 💾 Commit your changes (`git commit -m 'Add amazing feature'`)
4. 🚀 Push to the branch (`git push origin feature/amazing-feature`)
5. 🔀 Open a Pull Request

**中文** 🇨🇳
1. 🍴 Fork 本仓库
2. 🌿 创建你的特性分支 (`git checkout -b feature/amazing-feature`)
3. 💾 提交你的修改 (`git commit -m 'Add amazing feature'`)
4. 🚀 推送到分支 (`git push origin feature/amazing-feature`)
5. 🔀 打开一个 Pull Request

## 🎨 Code Style / 代码风格

**English** 🇬🇧
- 📏 Use 4 spaces for indentation
- 🏷️ Class names: PascalCase (e.g., `DERPool`)
- 🐍 Variable names: snake_case (e.g., `player_hp`)
- 🔧 Function names: snake_case (e.g., `get_value`)
- 💬 Add necessary comments

**中文** 🇨🇳
- 📏 使用 4 个空格缩进
- 🏷️ 类名：PascalCase（如 `DERPool`）
- 🐍 变量名：snake_case（如 `player_hp`）
- 🔧 函数名：snake_case（如 `get_value`）
- 💬 添加必要的注释

## 📝 Commit Message Format / 提交信息格式

**English** 🇬🇧
Format: `<type>: <subject>`

Types:
- ✨ `feat`: New feature
- 🐛 `fix`: Bug fix
- 📚 `docs`: Documentation changes
- 🎨 `style`: Code formatting (no code change)
- ♻️ `refactor`: Code refactoring
- 🧪 `test`: Adding tests
- 🔧 `chore`: Build process or tooling

Example:
```

✨ feat: add memory scanner detector
🐛 fix: resolve replay protection timeout issue
📚 docs: update README for v1.4.0

```

**中文** 🇨🇳
格式：`<type>: <subject>`

类型：
- ✨ `feat`: 新功能
- 🐛 `fix`: 修复 bug
- 📚 `docs`: 文档更新
- 🎨 `style`: 代码格式（不影响代码运行）
- ♻️ `refactor`: 代码重构
- 🧪 `test`: 添加测试
- 🔧 `chore`: 构建过程或工具

示例：
```

✨ feat: 添加内存扫描检测器
🐛 fix: 修复重放防护超时问题
📚 docs: 更新 v1.4.0 文档

```
